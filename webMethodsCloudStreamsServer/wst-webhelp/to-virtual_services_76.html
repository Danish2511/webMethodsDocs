CloudStreams will translate the POSTed request elements into query string parameters, in a root element. An incoming POST or PUT request cannot be translated into a GET or DELETE if the request has nested elements. For example:
(this is correct)
<person>
<lastName>Smith</lastName>
</person>
(this is incorrect)
<person>
<name>
<last>Smith</last>
</name>
</person>
If you want to send additional parameters as part of the request URL, you can transform this payload. To do this, you can use an XSLT file or a webMethods IS service call to add parameters before the request is sent to the native service. For more information, see The In Sequence Step (REST).
