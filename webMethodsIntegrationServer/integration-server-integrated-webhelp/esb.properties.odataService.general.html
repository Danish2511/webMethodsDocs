General Properties for OData Services
Property
Description
Permissions
Click to assign or view ACL permissions to an OData service.
Alias
Specifies an alternate name for the namespace name of the OData service.
Namespace
Displays the namespace name, which is the fully qualified name of the OData service on the Integration Server.
Use custom filter
Indicates whether or not to use custom filters instead of the built-in filters that Integration Server provides while using the $filter system query option.
Select...
To...
True
Use your custom filters in the $filter system query option. You can then specify custom filter queries as the value for the $filter parameter of the _retrieve and _update OData service implementations.
False
Use the built-in filters that Integration Server provides in the $filter system query option. This is the default.
