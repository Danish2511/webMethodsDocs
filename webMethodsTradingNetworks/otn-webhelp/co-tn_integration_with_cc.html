Using Command Central to Manage Trading Networks
Command Central provides you with a centralized management capability that enables you to manage the webMethods suite of products.
You can perform the following activities for Trading Networks using Command Central:
Manage Trading Networks servers from Command Central: Home > Instances > MWS_instance_name > Trading Networks > Configuration tab.
Manage Trading Networks properties from Command Central: Home > Instances > IS_instance_name > Trading Networks > Configuration tab > General Properties > Trading Networks properties.
Manage Trading Networks logs through the Integration Server that is hosting the Trading Networks instance.
Manage fixes by configuring the appropriate source repository for Trading Networks.
Perform template-based provisioning of Trading Networks instances.
Manage installation and uninstallation of Trading Networks assets by using provisioning asset commands.
Perform the following steps:
1. Select the Trading Networks assets to export.
For detailed instructions, see Selecting Trading Networks Assets For Export.
For detailed instructions on selecting assets for export, see webMethods Trading Networks Administrator’s Guide.
2. Build the exported assets.
For detailed instructions, see Building the Exported Assets.
For detailed instructions on building assets for export, see webMethods Trading Networks Administrator’s Guide.
3. Create the asset repository.
For detailed instructions, see Software AG Command Central Help.
4. Use the provisioning asset commands to install or uninstall Trading Networks assets.
For detailed instructions, see Software AG Command Central Help.
For details on how to the perform the Command Central operations, see Software AG Command Central Help.
Important:
The Trading Networks Server aliases that are created with special characters and white spaces on My webMethods server do not get listed on the Command Central user interface: Home > Instance > MWS_instance_name > Trading Networks > Configuration.
