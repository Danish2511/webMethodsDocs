To test partner connection
You can test an active connection for a trading partner to find out if the connection details are set appropriately.
Note: 
You cannot test a connection for the enterprise profile.
Test connection button does not appear in the Edit Delivery Method dialog box. You can test a connection only in the Add Delivery Method dialog box.
Test connection works only for the following protocols:
FTP
FTPS
HTTP
HTTPS
SFTP
SMTP
Primary E-mail
Primary FTP
Primary FTPS
Primary HTTP
Primary HTTPS
Primary SFTP
Secondary E-mail
Secondary FTP
Secondary FTPS
Secondary HTTP
Secondary HTTPS
Secondary SFTP
To test a partner connection
1. On the Partner Profiles page, select the profile to which you want to add delivery method.
2. Click the Delivery Settings tab.
3. Click Add Delivery Method, and select the Delivery Method of your interest.
4. Click Test connection.
5. Specify the following details on the Test your connection dialog box:
Field
Description
Document Content
Type the document content to test partner connection.
If you have webMethods Module for EDI installed, you can also submit EDI documents.
Content Type
Specify the content type. For example, XML, or EDI.
Sender
Specify your sender details. For HTTPS, Primary HTTPS, Secondary HTTPS, FTPS, Primary FTPS, Secondary FTPS.
6. Click Test to test an active connection for a trading partner to determine if the connection details are set up appropriately.
