You can perform the following tasks that act on all the files in a package as a unit:
Use this function
When you want to
Refer to
Create
Create a new package. Developers create packages using Designer. See webMethods Service Development Help for more information.
Creating a Package
Activate
Use a package that you manually moved into the Server/packages directory without having to restart the server.
Activating a Package
Reload
Reload the services in the package into memory without having to restart the server.
Reloading a Package
Enable
Enable a package that you previously disabled.
Enabling a Package
Disable
Disable access to a package without deleting it.
Disabling a Package
Delete
Delete all services and related files in a package.
Deleting a Package
Recover
Recover the services and related files from a package that you previously deleted.
Recovering a Package
Archive
Make a working copy of a package without making it generally available to others through a release. You might use this copy as a backup.
Archiving a Package
Copy
Copy a package from one server to another.
Archiving a Package
