Default Ports

The alias assigned to a default port on Integration Server cannot be changed. If you want to use a different alias for the port, protocol, and port type used by a default port, delete the port and recreate it with the new alias. To use a different alias for the DefaultPrimary port, you must designate another port as the primary port before deleting the DefaultPrimary port and recreating it with a new alias.
If you are running multiple Integration Servers on the same machine, the primary, diagnostic, and DefaultSecure port numbers on each server must be unique. For more information about running multiple Integration Server instances, see Running Multiple Integration Server Instances.
Note:
The DefaultSecure port uses the predefined DEFAULT_IS_KEYSTORE as the keystore alias and the DEFAULT_IS_TRUSTSTORE as the truststore alias which contain the default Integration Server keystore and truststore, respectively. Software AG strongly recommends that you change the keystore and truststore used with the DefaultSecure port as soon as you complete installation of Integration Server. Otherwise, your server will be vulnerable to anyone who knows the default keystore and truststore information installed with Integration Server.
