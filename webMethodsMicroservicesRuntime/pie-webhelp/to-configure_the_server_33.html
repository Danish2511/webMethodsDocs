Working with Extended Configuration Settings
There may be times when you want to view special server settings. The server configuration parameters are saved in the server.cnf file, however you can view them and edit them with Integration Server Administrator on the Settings > Extended page. For each parameter, Integration Server Administrator displays the current value and the default value. Integration Server Administrator also indicates whether a parameter requires a restart for changes to take effect.
On the Settings > Extended page, do one or more of the following:
Use Search to filter the displayed list of extended settings (i.e., server configuration parameters). Click Search to open the Extended settings search and do the following:
Search for one or more settings using some or all of the setting name. For example, in Key name enter “jms” and click Apply to list all of the settings that include “jms” in their name.
Search for settings that do not use the default value by selecting Settings changed form system defaults.
Display only your favorite settings by selecting the Hide unstarred settings check box.
To edit an extended setting, in the setting row click > Edit. In the Edit extended setting dialog box, provide the new value and click Save. If a server restart is required for the change to take effect, Integration Server Administrator displays a notification.
To add an extended setting, click . In the Add extended setting dialog box, provide the name of the setting and the value you want to use. Click Save to save the setting and value to the server.cnf file. You might need to add an extended setting as part of applying a fix to Integration Server.
To add multiple extended setting at one time, click > located above the table and select Batch add. In the Add multiple extended settings dialog box, enter each setting on its own line in a key=value format. Click Save to save the settings and values to the server.cnf file.
To reset a setting to its default value, in the setting row click > Reset to default.
To delete an extended setting, in the setting row click > Delete. Any system setting that you delete will be restored with their default value upon Integration Server restart.
To perform the same action on multiple settings at once, select the check boxes next to the setting names. Then click and select the action you want to apply.
Important:
Typically, you will use the Integration Server Administrator to set properties in the server.cnf file, but there may be times when you need to edit the file directly with a text editor. Before updating this file directly, be sure to shut down the Integration Server .
